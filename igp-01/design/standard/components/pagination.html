Pagination – Standard web pages

Use a pagination index for long documents that are best broken up over several pages.

Design options

A user can select a specific page within a document through either the "Next", "Previous", or (if present) the numeric pagination hyperlinks. This makes large content sections easier to read.

Type

  • Default
  • Simple pager
  • Simple pager justified

State

  • Active
  • Disabled

Size

  • Default
  • Small
  • Large
Appearance
  • Previous
  • 1 Go to Page 1
  • 2 Go to Page 2
  • Next
  • Previous
  • Next
HTML code 
<ul class="pagination pagination-sm pagination-lg">
  <li class="disabled"><a href="#" rel="prev">Previous</a></li>
  <li class="active"><a href="#">1 <span class="wb-inv">Go to Page 1</span></a></li>
  <li><a href="#">2 <span class="wb-inv">Go to Page 2</span></a></li>
  <li><a href="#" rel="next">Next</a></li>
</ul><ul class="pager pagination-sm pagination-lg">
  <li class="disabled previous"><a href="#" rel="prev">Previous</a></li>
  <li class="next"><a href="#" rel="next">Next</a></li>
</ul>

Coding considerations:

  • Add invisible text using <span class"wb-inv"> to make the links descriptive
  • Ensure there is a space between the pagination number and <span class"wb-inv"> to ensure screen readers correctly interpret the code
  • Use .previous and .next within the <li> elements to fully justify the <ul class="pager"> links

Disabled buttons

Disabled state (hidden "Previous" button).

Warning for active buttons

Only works with default type of pagination.

Correct use

  • Ensure the pagination is always left-aligned
  • Ensure the navigational links appear as a single row and don't wrap, unless there is a valid reason to do so
  • Use simple pager to create a simple pagination of only "Previous" and "Next" links
  • Use the correct state:
    • Use active to indicate the current page the user is on within the pagination
    • Use disabled to hide unnecessary navigation for example, a "Previous" button if the user is on "Page 1" of a content section.
  • Use the correct size:
    • Use default for typical pagination
    • Use small when the pagination appears in a small section of space. This reduces the button size to maximize the amount of buttons that appear in a single row
    • Use large when the pagination needs additional prominence on the page. Since the default buttons already create prominence and a call to action, there is limited value/need to increasing their size
