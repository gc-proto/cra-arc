Identify need for project

A project is initiated based on:

  • client and user needs
  • departmental priorities
  • web optimization planning

Gather existing data

Data is gathered in the planning phase, so it’s ready to be analyzed in the discovery phase.

The primary source of data is the Usability Performance Dashboard (UPD). It contains the following:

  • Call Centre data and trends
  • feedback tool
  • web analytics
  • Search Engine Optimization (SEO) data
  • Top task research
  • usability testing results and findings

Determine project scope

This portion of the planning phase establishes the project boundaries and identifies the Key Performance Indicators (KPIs), or measures of success, that will be assessed. It’s also when project timelines are determined.

  1. Identify top issues

    Content, call driver data, search patterns, user feedback, and website traffic are examined to identify the top issues that users are experiencing and that need to be addressed on the web pages.

  2. Establish timelines

    Determine the start and end dates for the tasks and activities associated with the project.

  3. Set KPIs

    For every project, KPIs are set.

    Rate of findability and task success

    A rate of 80% in findability and task success (completion)

    An increase in success from initial baseline to validation testing

    A 20% point increase (from initial baseline to validation testing) in terms of findability and task success (completion)

    The following are considered when determining whether KPIs have been met:

    Ease of use
    Users rank how easy or difficult it is to complete a task.
    System Usability (SUS)
    Users participate in this 10-item questionnaire, administered at the end of a usability test. The aim of the questionnaire is to capture the participant’s overall experience with the test pages and measure the usability of the test pages against industry-established metrics.
    Time on task
    How long it takes the user to complete a task.

Engage stakeholders and project team

All key stakeholders from the various departments and specialties are included in the planning process along with the project team.

Kickoff meeting

The goal of the kickoff meeting is to ensure all key stakeholders from the various departments and specialties are aligned and form one unified team for the project.

Some of the items that should be confirmed during the meeting:

  • intended target audience and their needs
  • problems that need to be solved
  • whether there is any current data related to pages (existing statistics or research)
  • timelines/deadlines
  • business goals from the different departmental areas
  • whether the top “problem statements” for each partner area are in alignment with one another (high level issue each partner area feels is a cause for concern)
  • business, user, or technology constraints (high-season volumes, planned release cycles, accessibility, standards, etc.)
  • technical limitations that need to be dealt with
