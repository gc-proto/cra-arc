Labels – Standard web pages

Use labels to call attention to a section of content by adding short, text-based bits of supporting information. Labels are visual indicators that assign a tag to items such as events, products, programs or services. They provide additional context about the adjacent content.

On this page

  • Design options
  • Complementary components and functions

When to use

Use labels to:

  • signal that an item belongs to a category (for example, to show if some programs are open, some closed, and some will open soon)
  • label or stamp items with additional information, like the date an item was updated, or the closure of an area
  • show different stages of progress in a list of items (for example, to show which vaccines are under review and which ones are approved)

Design options

Use the appropriate label to describe and highlight text-based (non-numerical) information.

Correct use

  • Ensure labels are placed very close to the item they apply to
  • Use the correct type of label
    • Use default to give additional information about an item, like adding dates to a list of items
    • Use info to tag an item with a value, like identifying a new blog post. "New" labels should be temporary
    • Use primary to signal an important information about an item, like when a new program is not yet open but will be soon
    • Use success to signal an item is open, available or approved
    • Use warning when you want to bring attention to an item to warn people of a change, like if a program is ending soon
    • Use danger when an item is no longer available, like when a program has ended or an area is closed

Incorrect use

  • Don't use long labels or full sentences - keep them short
  • Don't use labels that have overlapping meaning
  • Don't use labels to add style to text
  • Do not use in alerts or buttons
  • Do not use to highlight unread items − use Badges
  • Do not highlight large sections of text − use Alerts
